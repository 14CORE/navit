75 /*
76  When a message is supplied to espeak_synth, the request is buffered and espeak_synth returns. When the message is really processed, the callback function will be repetedly called.
77 
78 
79  In RETRIEVAL mode, the callback function supplies to the calling program the audio data and an event list terminated by 0 (LIST_TERMINATED).
80 
81  In PLAYBACK mode, the callback function is called as soon as an event happens.
82 
83  For example suppose that the following message is supplied to espeak_Synth:
84  "hello, hello."
85 
86 
87  * Once processed in RETRIEVAL mode, it could lead to 3 calls of the callback function :
88 
89  ** Block 1:
90  <audio data> +
91  List of events: SENTENCE + WORD + LIST_TERMINATED
92 
93  ** Block 2:
94  <audio data> +
95  List of events: WORD + END + LIST_TERMINATED
96 
97  ** Block 3:
98  no audio data
99  List of events: MSG_TERMINATED + LIST_TERMINATED
100 
101 
102  * Once processed in PLAYBACK mode, it could lead to 5 calls of the callback function:
103 
104  ** SENTENCE
105  ** WORD (call when the sounds are actually played)
106  ** WORD
107  ** END (call when the end of sentence is actually played.)
108  ** MSG_TERMINATED
109 
110 
111  The MSG_TERMINATED event is the last event. It can inform the calling program to clear the user data related to the message.
112  So if the synthesis must be stopped, the callback function is called for each pending message with the MSG_TERMINATED event.
113 
114  A MARK event indicates a <mark> element in the text.
115  A PLAY event indicates an <audio> element in the text, for which the calling program should play the named sound file.
116 */

481 /* Note: The espeak_VOICE structure is used for two purposes:
482  1. To return the details of the available voices.
483  2. As a parameter to espeak_SetVoiceByProperties() in order to specify selection criteria.
484 
485  In (1), the "languages" field consists of a list of (UTF8) language names for which this voice
486  may be used, each language name in the list is terminated by a zero byte and is also preceded by
487  a single byte which gives a "priority" number. The list of languages is terminated by an
488  additional zero byte.
489 
490  A language name consists of a language code, optionally followed by one or more qualifier (dialect)
491  names separated by hyphens (eg. "en-uk"). A voice might, for example, have languages "en-uk" and
492  "en". Even without "en" listed, voice would still be selected for the "en" language (because
493  "en-uk" is related) but at a lower priority.
494 
495  The priority byte indicates how the voice is preferred for the language. A low number indicates a
496  more preferred voice, a higher number indicates a less preferred voice.
497 
498  In (2), the "languages" field consists simply of a single (UTF8) language name, with no preceding
499  priority byte.
500 */
